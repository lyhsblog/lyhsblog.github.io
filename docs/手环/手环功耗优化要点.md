# 手环固件功耗优化要点

本文归纳 nRF52 + LIS3DH + MAX30102 类方案中常见的 **软件侧省电手段**，并与仓库示例工程 **`examples/nrf52-lis3dh-max30102/`** 的 **当前实现** 对照。电流须以 **实机电流表** 与各芯片 **数据手册** 为准。

---

## 0. 睡眠策略：本方案采用「浅睡」

手环示例与推荐产品路径采用 **System ON 下的 CPU 浅睡**，主循环使用 **`__WFI()`**：

- **CPU** 停跑指令，**RAM 与外设寄存器保持**；  
- **LFCLK、RTC（分钟 COMPARE + 按需 TICK）、GPIOTE（LIS3DH INT1）** 仍可唤醒；  
- **TWI / 传感器** 可按策略保持上电或分级关闭。

**不推荐**在本场景（依赖 **片内 RTC 整分钟任务** + **I²C 传感器**）下默认进入 **System OFF（深睡）**：深睡唤醒源与 RTC 行为需对照 **Product Specification**，且常需 **冷启动式重新 init**，与「每分钟落盘 + 条件心率」架构冲突；深睡更适合 **摘下仓、长按关机** 等子状态单独设计。

下文中的 **「优化」** 均指在 **浅睡前提** 下减少 **无效唤醒、缩短 RUN 时间、关传感器与总线**。

## 2. TWI（I²C）不用时 uninit / disable

### 含义

在一段时间内 **不再访问总线** 时，调用 **`nrf_drv_twi_disable()`** / **`nrf_drv_twi_uninit()`**。

### 为何有效

减少 TWI 外设与 SCL/SDA 在空闲时的 **静态与漏电**。

### 实施注意

重新通信前 **`init` + `enable`**；多器件共总线时 **不能** 在另一器件仍需通信时关主机。

### 与示例工程的关系

**待办**：仅在 **RTC 分钟 + GPIOTE 等待**、且 **确认不再读传感器** 的时段，可评估 **uninit TWI**；在 **开 TICK 计步 / 启动心率状态机前** 再 init。

---

## 3. LIS3DH 降低 ODR / 低功耗模式

### 含义

通过寄存器降低 **ODR**、使用 **LP**；**静止** 与 **走动/爆发采样** 间 **两档切换**。

### 为何有效

降低加速度计内部 **采样与转换** 活动电流。

### 实施注意

**INT1 运动检测** 对 **ODR/滤波** 有下限，须按 **ST 手册** 验证；爆发窗口内可与 **RTC TICK 节拍** 对齐 **临时提高 ODR**。

### 与示例工程的关系

**待办**：爆发结束关 TICK 时，可同步 **写回更低 ODR**；静止段仅依赖 **INT1** 时维持 **最低可靠 ODR**。

## 4. MCU：浅睡（WFI）与深睡（SYSTEM OFF）

### 浅睡：`__WFI()` + System ON（**当前采用**）

- 适合 **RTC 分钟**、**门控 TICK**、**GPIOTE**、**按需 I²C**；  
- 省电主要来自 **减少唤醒频率** 与 **缩短 RUN**，而非关断内核。

### SYSTEM OFF（深睡）

- 唤醒源与 **片内 RTC 分钟闹钟** 的兼容性 **不可默认假设**，见 §0；  
- 适合 **明确子状态**（如仓内存储、彻底关机），需单独 **状态机与唤醒脚** 设计。

### 选型小结

| 需求 | 倾向 |
|------|------|
| **分钟 RTC + 传感器 I²C** | **浅睡 + 外设分级关断**（当前示例方向） |
| **长时间无业务、可丢 RAM** | 单独评估 **SYSTEM OFF** + **GPIO/充电等唤醒** |

## 5. 光学心率（MAX30102）

- **测量间隙 Shutdown**，避免 LED **mA 级** 常开。  
- **示例已实现**：心率由 **RTC TICK 状态机** 推进，**无** 测量窗内 **`nrf_delay_ms` 长阻塞**。  
- **可选**：MAX30102 **INT**（FIFO 水位）进一步减少 **无效轮询**（当前为每 TICK `collect_fifo_red`）。
