# 手环固件功耗优化要点

本文归纳 nRF52 + LIS3DH + MAX30102 类方案中常见的 **软件侧省电手段**，与示例工程 `examples/nrf52-lis3dh-max30102/` 中的策略（如 **静止关 RTC TICK**、心率 **Shutdown**）可对照使用。具体电流须以 **实机电流表** 与 **数据手册** 为准。

---

## 1. TWI（I²C）不用时 uninit / disable

### 含义

在一段时间内 **不再访问总线**（不读 LIS3DH、不读 MAX30102）时，调用 **`nrf_drv_twi_disable()`** / **`nrf_drv_twi_uninit()`**，释放 TWI 外设与相关引脚配置。

### 为何有效

- TWI 外设与引脚保持 **主机空闲** 状态时仍有 **静态与漏电**；  
- SCL/SDA 为 **开漏 + 上拉** 时，不当配置可能增加 **翻转或浮空漏电**。

### 实施注意

- 下次通信前必须 **重新 `init` + `enable`**，增加 **数十微秒～毫秒级** 延迟，需避免与硬实时 ISR 冲突；  
- **同一总线多器件** 时，只要 **任一器件** 还要通信，就不宜关总线；  
- uninit 后可将引脚设为 **输入 + 明确上拉/下拉** 或 **模拟**，避免 **浮空**。

### 与示例工程的关系

当前示例在 **计步爆发窗口** 与 **潜在读传感器** 阶段需要 TWI；若在 **仅 RTC 分钟任务 + 等待 GPIOTE** 的长窗口内 **完全不读 I²C**，可评估 **短时 uninit TWI**，在 **打开计步 / 测心率前** 再 init。

## 2. LIS3DH 降低 ODR / 低功耗模式

### 含义

通过 **CTRL_REG1** 等寄存器降低 **输出数据速率（ODR）**，并启用数据手册中的 **LP（低功耗）** 位；必要时在 **静止** 与 **走动** 间切换 **两档配置**。

### 为何有效

加速度计内部 **采样、转换、滤波** 频率越高，电流越大；**ODR 降低**直接减少内部活动。

### 实施注意

- **运动中断（INT1）** 路径往往仍要求 **最低 ODR / 滤波** 条件，需对照 **ST AN3308 / 数据手册** 验证 **阈值 + 持续时间** 是否仍可靠；  
- 计步算法若依赖 **固定采样间隔**（如 25 ms），降 ODR 后需 **重标定** 或 **仅在「爆发窗口」临时提高 ODR**；  
- **读 OUT_X/Y/Z 与中断配置** 可分两套：**静止低 ODR + 走动高 ODR**，由状态机切换。

### 与示例工程的关系

示例在 **运动爆发后** 限时开 **RTC TICK** 读加速度；LIS3DH 侧可同步 **提高 ODR** 以匹配算法，**爆发结束** 后 **写回低 ODR**，与 **关 TICK** 一起降低整体电流。

## 3. MCU：WFI / SLEEP 与 SYSTEM OFF

### `__WFI()` / 浅睡眠（SYSTEM ON）

- CPU 停止执行指令，**多数外设仍可运行**（RTC、GPIOTE、TWI 等）；  
- 由 **中断** 唤醒；适合 **RTC 分钟节拍 + TICK 门控 + GPIO 运动中断** 等架构；  
- 示例主循环 **`__WFI()`** 即属此类，省电程度 **中等**，集成简单。

### SYSTEM OFF（深睡眠）

- **最深档**之一：内核与大部分逻辑 **关断**，**RAM 是否保留** 依配置与型号而定；  
- 仅 **有限唤醒源**（如部分 **GPIO latch**、**复位** 等，**须查具体 nRF52 型号与 SoftDevice 文档**）；  
- 唤醒后常接近 **重新引导应用层**：全局变量、外设寄存器 **需重新初始化**；  
- **调试困难**，适合 **产品化阶段** 在 **明确唤醒策略** 后启用。

### 选型建议

| 需求 | 倾向 |
|------|------|
| 要 **分钟 RTC**、**I²C 随时可起** | **WFI + 关不必要中断源**（如关 TICK） |
| **长时间无业务**（如摘下、关机态） | 评估 **SYSTEM OFF** + **按键/充电唤醒** |

---

## 4. 光学心率（MAX30102）侧（简要）

- **测量间隙** **`MODE_CONFIG` Shutdown** 或关 LED，避免 **mA 级** 持续电流；  
- **FIFO + 中断** 可减少 CPU **轮询**；示例中心率窗口内仍用 **`nrf_delay_ms`**，量产可改为 **事件驱动** 以缩短 **RUN 时间**。

---

## 5. 推荐实施顺序（工程上）

1. **关不需要的周期中断**（如静止时 **关 RTC TICK**，仅保留 **COMPARE0** 与 **GPIOTE**）。  
2. **传感器测完即关**：MAX30102 **Shutdown**；LIS3DH **降 ODR**。  
3. **总线空闲** 再评估 **TWI uninit**。  
4. **主循环** 用 **`WFI`**，确认 **唤醒源列表** 最小化。  
5. 在 **电流曲线达标** 后，再评估 **SYSTEM OFF** 与 **RAM 保留** 方案。
