# 手环固件样例：RAM 队列 + 周期落 Flash

本文说明一段面向 **nRF52840**（Nordic SDK、`nrf_drv_*` 风格）的示意代码：在 **GPIO 中断**里采集步数与心率，先写入 **RAM 环形队列**；由 **RTC 比较中断**每分钟触发一次，将队列中数据 **批量写入 Flash**（`nrf_fstorage`），主循环以 `__WFE()` 等待事件以降低功耗。

下文中的代码为文档化整理，与工程内实际文件名无关；集成到真实工程时需补齐 SoftDevice、fstorage 事件处理、时钟源等配套初始化。

## 功能概览

| 模块 | 作用 |
|------|------|
| `data_t` | 单条记录：时间戳、步数、心率 |
| RAM 队列 | 中断里快速 `push`，避免在 ISR 内长时间写 Flash |
| GPIOTE | 传感器（示意：引脚 17）边沿触发采样 |
| RTC | 约每分钟触发一次，调用 `write_flash_batch()` |
| `nrf_fstorage` | 在 `0x3F0000`～`0x3FFFFF` 区间顺序写入/按页擦除 |

## 数据结构

```c
typedef struct {
    uint32_t timestamp;
    uint16_t steps;
    uint8_t  heart_rate;
} data_t;
```

- `timestamp`：样例中在中断里写 `0`，实际应改为 **RTC 计数或 Unix 时间**（需先完成 RTC 日历或 tick 换算）。
- `steps` / `heart_rate`：样例通过 `read_steps_sensor()`、`read_hr_sensor()` 占位返回固定值，真实设备应对接计步与外设心率算法或传感器驱动。

## 配置常量

| 宏 | 含义 |
|----|------|
| `MAX_RAM_QUEUE` | RAM 环形队列容量（条数），队列满时当前实现未写入（可改为覆盖最旧或丢弃策略） |
| `FLASH_START_ADDR` | Flash 数据区起始地址，须与 **链接脚本 / 分区表** 一致，避免覆盖 Bootloader 或应用 |
| `FLASH_PAGE_SIZE` | nRF52840 常见为 4096 字节；擦除粒度按页 |
| `fstorage.start_addr` / `end_addr` | 与 `FLASH_START_ADDR` 及末尾地址对齐，且落在合法 Flash 范围内 |

## 程序流程

### 1. 传感器中断（高优先级、短路径）

`init_sensor_interrupt()` 初始化 GPIOTE，引脚 **17** 配置为下降沿（`HITOLO`）、上拉。`sensor_irq_handler` 中：

1. 组装 `data_t`（时间戳、步数、心率）。
2. 调用 `push_ram(&d)` 写入环形队列。

`push_ram` 使用 `(head + 1) % MAX_RAM_QUEUE` 与 `tail` 比较判断是否已满；满时当前代码块为空，**不会入队**，生产环境应明确策略（丢新、丢旧或阻塞，一般不阻塞 ISR）。

### 2. 周期落盘（RTC）

`init_rtc()` 使用 RTC 实例 0，默认配置下若 tick 为 1 Hz，则 `nrf_drv_rtc_cc_set(..., 60, ...)` 表示约 **60 秒** 触发一次比较中断。`rtc_handler` 在 `NRF_DRV_RTC_INT_COMPARE0` 时调用 `write_flash_batch()`。

若实际 RTC 预分频与 tick 不同，需按数据手册重新计算 **compare 值**，避免间隔与预期不符。

### 3. Flash 批量写入

`write_flash_batch()` 在 `ram_tail != ram_head` 时循环：

1. 取 `ram_queue[ram_tail]` 指针。
2. 若当前 `flash_addr` 相对起始地址对齐到 **新页起点**，先 `nrf_fstorage_erase(..., 1, ...)` 擦一页。
3. `nrf_fstorage_write` 写入一条 `sizeof(data_t)`。
4. `flash_addr` 递增，`ram_tail` 环形前进。

这样可在同一页内连续写入多条记录，跨页时先擦后写。

### 4. 主函数

`main` 中调用 `nrf_fstorage_init`（使用 `nrf_fstorage_sd` 后端）、`init_rtc()`、`init_sensor_interrupt()`，随后 `while(1) { __WFE(); }` 进入低功耗等待。

## 样例源码（整理版）

以下为与说明对应的完整示意程序，便于复制到工程或对照阅读。

```c
#include <stdint.h>
#include <string.h>
#include "nrf_drv_rtc.h"
#include "nrf_drv_gpiote.h"
#include "nrf_delay.h"
#include "nrf_fstorage.h"
#include "nrf_fstorage_sd.h"
#include "nrf_log.h"

// ---------- 配置 ----------

#define MAX_RAM_QUEUE 128   // RAM 缓存条数
#define FLASH_START_ADDR 0x3F0000  // Flash 数据存储起始地址
#define FLASH_PAGE_SIZE 4096       // 内置 Flash 页大小（nRF52840）

typedef struct {
    uint32_t timestamp;
    uint16_t steps;
    uint8_t heart_rate;
} data_t;

static data_t ram_queue[MAX_RAM_QUEUE];
static volatile uint8_t ram_head = 0;
static volatile uint8_t ram_tail = 0;
static uint32_t flash_addr = FLASH_START_ADDR;

// ---------- fstorage 配置 ----------
NRF_FSTORAGE_DEF(nrf_fstorage_t fstorage) =
{
    .evt_handler = NULL,
    .start_addr = 0x3F0000,
    .end_addr   = 0x3FFFFF,
};

// ---------- RAM 队列操作 ----------
static void push_ram(data_t *data) {
    uint8_t next_head = (ram_head + 1) % MAX_RAM_QUEUE;
    if(next_head != ram_tail) {  // 队列未满
        ram_queue[ram_head] = *data;
        ram_head = next_head;
    } else {
        // 队列满可选择丢弃或覆盖
    }
}

// ---------- 模拟传感器读取 ----------
uint16_t read_steps_sensor(void) { return 1; }
uint8_t read_hr_sensor(void) { return 75; }

// ---------- 传感器中断 ----------
void sensor_irq_handler(nrf_drv_gpiote_pin_t pin, nrf_gpiote_polarity_t action) {
    data_t d;
    d.timestamp = 0; // 可使用 RTC 获取真实时间
    d.steps = read_steps_sensor();
    d.heart_rate = read_hr_sensor();
    push_ram(&d);
}

// ---------- Flash 批量写入 ----------
void write_flash_batch(void) {
    // 每次写一页或队列中所有数据
    while(ram_tail != ram_head) {
        data_t *d = &ram_queue[ram_tail];

        // 如果当前页满，则先擦除
        if((flash_addr - FLASH_START_ADDR) % FLASH_PAGE_SIZE == 0) {
            nrf_fstorage_erase(&fstorage, flash_addr, 1, NULL);
        }

        nrf_fstorage_write(&fstorage, flash_addr, d, sizeof(data_t), NULL);
        flash_addr += sizeof(data_t);
        ram_tail = (ram_tail + 1) % MAX_RAM_QUEUE;
    }
}

// ---------- RTC 每分钟触发 ----------
static void rtc_handler(nrf_drv_rtc_int_type_t int_type) {
    if(int_type == NRF_DRV_RTC_INT_COMPARE0) {
        write_flash_batch();
    }
}

// ---------- 初始化 RTC ----------
static const nrf_drv_rtc_t rtc = NRF_DRV_RTC_INSTANCE(0);

void init_rtc(void) {
    nrf_drv_rtc_config_t rtc_cfg = NRF_DRV_RTC_DEFAULT_CONFIG;
    nrf_drv_rtc_init(&rtc, &rtc_cfg, rtc_handler);

    // 每分钟触发一次
    uint32_t ticks = 60; // RTC tick = 1 Hz
    nrf_drv_rtc_cc_set(&rtc, 0, ticks, true);

    nrf_drv_rtc_enable(&rtc);
}

// ---------- 初始化传感器中断 ----------
void init_sensor_interrupt(void) {
    nrf_drv_gpiote_init();
    nrf_drv_gpiote_in_config_t config = GPIOTE_CONFIG_IN_SENSE_HITOLO(true);
    config.pull = NRF_GPIO_PIN_PULLUP;

    nrf_drv_gpiote_in_init(17, &config, sensor_irq_handler); // 假设引脚17
    nrf_drv_gpiote_in_event_enable(17, true);
}

// ---------- 主程序 ----------
int main(void) {
    nrf_fstorage_init(&fstorage, &nrf_fstorage_sd, NULL);

    init_rtc();
    init_sensor_interrupt();

    while(1) {
        __WFE(); // 低功耗等待中断
    }
}
```

## 接入真实工程时的注意点

1. **SoftDevice 与 fstorage**：使用 `nrf_fstorage_sd` 时，通常需先启用 SoftDevice，并在 `evt_handler` 或主循环中 **轮询 `nrf_fstorage_process()`**（或按 SDK 示例在调度器里处理完成事件）。样例中 `evt_handler` 为 `NULL`、且主循环仅有 `__WFE()`，在真实工程中 **必须** 按 Nordic 文档补全，否则擦写可能无法完成。
2. **写队列与写 Flash 的并发**：`ram_tail` 在 `write_flash_batch` 中修改，ISR 中修改 `ram_head`。若目标平台对多字节读写非原子，需评估是否用 **关中断临界区** 或 **双缓冲**，避免撕裂。
3. **`write_flash_batch` 在中断里执行**：RTC 回调中执行擦写可能耗时较长，一般更稳妥的是 **仅置标志位**，在 **主循环或低优先级上下文** 中执行 Flash 操作，以免阻塞其他中断。
4. **Flash 地址耗尽**：`flash_addr` 递增到 `end_addr` 后需 **换区、循环缓冲或上报上位机**，避免越界。
5. **链路与功耗**：使用 `__WFE()` 时通常需配合 `SEV`/事件置位机制；若使用 SoftDevice，往往用 `sd_app_evt_wait()` 等 API，与裸 `__WFE()` 二选一或按官方示例组合。

以上几点不影响理解本样例的 **数据路径设计**（中断采样 → RAM 队列 → 定时批量落盘），但在量产固件中需要逐项落实。
