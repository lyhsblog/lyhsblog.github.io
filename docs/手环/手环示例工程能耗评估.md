# nRF52 示例工程能耗评估（`examples/nrf52-lis3dh-max30102`）

本文对 **当前仓库内示例代码** 做 **定性 + 量级框架** 评估，**不替代** 板级电流表与 Nordic/ST/Maxim 数据手册。硬件（晶振、LDO、漏电流）、是否启用 **SoftDevice**、Flash 写入等都会显著改变结果。

---

## 1. 示例在做什么（与功耗直接相关）

| 模块 | 行为 |
|------|------|
| **LFCLK + RTC2** | 常开；**COMPARE0** 每 **60 s** 一次；**TICK** 仅在 **运动爆发后 `MOTION_BURST_SEC`（默认 12 s）** 内开启（约 **40 Hz**） |
| **TWI** | **`i2c_bus_init` 后保持 enable**，静止时仍占用外设（**未**按「空闲 uninit」优化） |
| **LIS3DH** | **`lis3dh_init_motion_interrupt`** 后持续工作在中等 ODR + 中断路径（**未**在静止时降 ODR） |
| **MAX30102** | 平时 **Shutdown**；心率窗口内 **HR 模式 + LED**，约 **4 s**（`HR_MEASURE_WINDOW_MS`），其间 **`nrf_delay_ms` 轮询 FIFO**（CPU 多为 RUN） |
| **CPU** | 主循环 **`__WFI()`**；由 **GPIOTE、RTC TICK（爆发期）、RTC COMPARE0** 唤醒 |

---

## 2. 工况划分（便于分项估算）

### A. 长时间静止（无走动、无 INT1、本分钟不测心率）

- **唤醒源**：约 **每 60 s** 一次 **RTC COMPARE0**（处理 `on_minute_tick`，多数分钟仅 **stub 落盘** + 可能 **不测** MAX30102）。  
- **RTC TICK**：**关**。  
- **平均电流**主要由 **常开外设** 决定：  
  - nRF：**LFCLK**、**RTC**、**GPIOTE**、**TWI 外设**、可能的 **HFCLK 策略**（视 SDK 与睡眠实现）；  
  - **LIS3DH**：配置为运动检测时的 **持续采样/中断路径**（通常 **显著大于** 纯 RTC 待机）。  
- **定性**：相对「仅 RTC 闹钟的手表」**更费电**，因为 **加速度计未关、TWI 未关**。

### B. 静止但本分钟触发心率（步数 >100 或低活动满 5 分钟）

- 在 **A** 的基础上，每分钟任务内增加 **约 4 s** 的 **MAX30102 开灯 + MCU RUN（delay 轮询）**。  
- **定性**：该分钟平均电流 **明显高于** 纯静止分钟；**占空比** ≈ 4 s / 60 s 时，心率窗口对 **日平均** 的贡献可用 **时间加权** 粗算。

### C. 走动（反复 INT1 + 12 s 爆发窗口）

- **RTC TICK 打开** 约 **12 s**（可因再次 INT1 **续期**），约 **每 25 ms** 唤醒一次：  
  **I²C 读 LIS3DH** + **step_counter** 计算。  
- **定性**：这段时间平均电流 **高于 A**，接近 **连续低占空比传感器采集**；若 **持续走动**，爆发窗口可 **长时间续期**，整体更接近 **连续 40 Hz 采样** 量级。

---

## 3. 量级思路（如何自己算，不写死 µA）

1. **用电流表** 分别测：  
   - 仅 nRF + LFCLK + RTC（可临时关掉 LIS3DH 电源或 uninit TWI 做对比）；  
   - 加上 **LIS3DH 当前配置**；  
   - 再加上 **MAX30102 HR 窗口**（夹表在电池回路）。  
2. **时间加权**：  
   - `I_day ≈ I_quiet × t_quiet/T + I_burst × t_burst/T + I_hr × t_hr/T`  
   - 其中 `t_hr` = 每天实际测心率总时长，`t_burst` = 每天处于 12 s 爆发续期状态的总时长。  
3. **电池**：`续航(h) ≈ 容量(mAh) / I_avg(mA)`（未计自放电与低温）。

---

## 4. 与「已做省电」的关系

示例已实现：

- **静止关 RTC TICK**（去掉 **40 Hz 无差别唤醒**），相对「始终 TICK 计步」**明显有利**；  
- **MAX30102 非测量 Shutdown**、心率 **短时窗口**，相对 **常亮 PPG** **大幅有利**。

仍未做（文档 [手环功耗优化要点](./手环功耗优化要点.md) 已列）：

- **TWI 空闲 uninit**；  
- **LIS3DH 静止降 ODR / 更深睡眠**；  
- **SYSTEM OFF**；  
- 心率窗口内 **减少 busy-wait**（改中断/FIFO 水位唤醒）。

---

## 5. 结论（一句话）

**当前示例**在 **架构上** 已避开最严重的 **常开 25 ms 节拍** 与 **常亮 PPG**，**静止平均电流仍主要由「LIS3DH 常开 + TWI 常开 + LFCLK/RTC」决定**；**走动**与 **心率分钟** 会再拉高 **时间加权平均**。**精确 mAh/天 必须以实机分项电流 × 占空比计算**，本页仅提供 **拆解框架**。
